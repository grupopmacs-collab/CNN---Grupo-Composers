Resumen del Proyecto

Desarrollamos un sistema de detección de fraude en transacciones utilizando Redes Neuronales Convolucionales (CNN) que analiza imágenes de comprobantes para identificar patrones sospechosos. El modelo fue entrenado con el dataset MNIST adaptado para simular transacciones legítimas y fraudulentas.

Objetivo

Crear un sistema capaz de detectar transacciones fraudulentas en tiempo real con alta precisión, minimizando los falsos positivos que afectan la experiencia del usuario.

Metodología

Implementamos el proceso CRISP-DM para el desarrollo del modelo, incluyendo comprensión del negocio, preparación de datos, modelado con CNN, evaluación y despliegue.

Resultados

El modelo alcanzó una precisión del 97.4% en validación, con un AUC de 0.9958, superando los criterios de éxito establecidos para el proyecto.

Arquitectura del Modelo

Nuestra CNN utiliza una arquitectura optimizada para la detección de patrones en imágenes de transacciones:

Capas Convolucionales

3 bloques convolucionales con activación ReLU, normalización por lotes y capas de pooling para extracción de características.

Filtros: 32 → 64 → 128

Capas Densas

Dos capas fully connected con dropout para prevenir overfitting y una capa de salida con activación sigmoid para clasificación binaria.

Optimización

Optimizador Adam con learning rate de 0.001, función de pérdida binary crossentropy y métricas de precisión, recall y AUC.

Proceso de Desarrollo - 6 Fases

1 Fase 1: Entendimiento del Negocio

Definición del problema, objetivos y criterios de éxito. Análisis de requerimientos para la detección de fraude en transacciones.

2 Fase 2: Preparación de Datos

Recolección y preprocesamiento del dataset MNIST. Balanceo de clases y transformación de imágenes a formato compatible con CNN.

3 Fase 3: Modelado

Diseño y entrenamiento de la arquitectura CNN. Optimización de hiperparámetros y validación del modelo.

4 Fase 4: Evaluación

Análisis de métricas de rendimiento y comparación con criterios de éxito. Validación en conjunto de prueba.

5 Fase 5: Métricas y KPIs

Evaluación exhaustiva del modelo contra criterios de negocio. Análisis de impacto económico y ROI del sistema.

6 Fase 6: Despliegue

Implementación del sistema de detección en tiempo real con interfaz para subida de imágenes y análisis de resultados.

Dataset

Utilizamos MNIST dataset adaptado, donde los dígitos 0-4 representan transacciones legítimas y 5-9 transacciones fraudulentas.

Infraestructura

Entrenamiento en Google Colab con GPU Tesla T4. Modelo optimizado para inferencia en tiempo real con baja latencia.

Especificaciones CNN

3 capas convolucionales con 32, 64 y 128 filtros respectivamente. 224 filtros totales para extracción de características.
